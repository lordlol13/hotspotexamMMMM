# Emergent Microscopy: Image Hotspot Quiz & Slides Platform 🔬🎯

Интерактивная платформа для работы с гистологическими слайдами высокого разрешения и проведения экзаменов по микроскопии. Преподаватели могут загружать слайды (SVS/TIFF), размечать интерактивные области (hotspots) с привязанными вопросами или медиафайлами, а студенты — изучать микропрепараты через масштабируемый просмотрщик, сдавать экзамены и отслеживать свой прогресс на дашборде.

---

## Основные возможности

### 🔬 Визуализация микроскопии высокого разрешения (SVS)
* Поддержка сверхкрупных файлов слайдов (**10GB+**) в форматах **SVS, TIFF, PNG, JPG, WEBP**.
* Использование библиотеки **OpenSlide** на бэкенде для динамической генерации тайлов Deep Zoom "на лету".
* Фронтенд-просмотрщик на базе **OpenSeadragon** с плавным панорамированием, масштабированием и навигационной картой.

### ✍️ Редактор интерактивных аннотаций
* Рисование областей поверх слайда: **прямоугольники, круги, полигоны, произвольное рисование (Freehand)**.
* Геометрия областей сохраняется в базе данных **в процентах**, обеспечивая корректное отображение на любых экранах и масштабах.
* Привязка контента к областям:
  * **Экзаменационный вопрос** (интеграция с движком тестирования).
  * **Текстовое пояснение**.
  * **YouTube-видео, аудиофайлы, PDF-документы** или **внешние веб-ссылки**.

### 📝 Экзаменационный движок (Exam Engine)
* Поддержка **5 типов вопросов**:
  1. Один вариант ответа (Single Choice).
  2. Несколько вариантов (Multiple Choice / Checkbox).
  3. Верно/Неверно (True/False).
  4. Краткий ответ (Short Answer).
  5. Эссе / Свободный ответ (Essay).
* Таймер экзамена с автоматической отправкой ответов при истечении времени.
* Настройки ограничений: количество попыток, лимиты времени, проходной балл, выдача индивидуальных пересдач (retakes).
* Автоматическая оценка с сохранением результатов в базе данных.

### 📊 Дашборды аналитики (Power BI Style)
* Визуализация данных с помощью адаптивных графиков **Recharts**.
* **Панель студента**:
  * Общий прогресс и средний балл.
  * История попыток и хронологический тренд оценок.
* **Панель преподавателя**:
  * Статистика групп и средний процент сдачи.
  * Рейтинговая таблица студентов (leaderboard).
  * Сложность вопросов (какой процент студентов ответил верно).

### 🔐 Безопасность и управление доступом
* Ролевая авторизация (Администратор, Преподаватель, Студент).
* Аутентификация на базе **JWT Access & Refresh токенов**.
* Форма регистрации студента с полями: имя, фамилия, ID студента, факультет, курс, группа, email, пароль.
* Подтверждение почты и сброс пароля.

## Архитектура и технологии

### Бэкенд
* **FastAPI** — высокопроизводительный веб-фреймворк.
* **PostgreSQL** — надежная реляционная база данных.
* **SQLAlchemy (V2.0)** — современная ORM.
* **Alembic** — автоматическое управление миграциями бэкенда на лету.
* **OpenSlide Python Wrapper & Pillow** — обработка гистологических изображений.

### Фронтенд
* **React 18** (Vite + TypeScript).
* **Material UI (MUI)** — премиальный темный интерфейс с элементами glassmorphism.
* **OpenSeadragon** — масштабируемый просмотрщик изображений сверхвысокого разрешения.
* **Recharts** — интерактивные графики и диаграммы.

## Быстрый запуск (Docker Compose)

Для запуска всей платформы локально выполните следующие шаги:

1. Убедитесь, что у вас установлен Docker и запущен Docker Desktop.
2. Склонируйте репозиторий и перейдите в его корень.
3. Запустите сборку и старт контейнеров:
   ```bash
   docker compose up --build
   ```
4. Доступ к веб-интерфейсам:
   * **Панель управления (Frontend)**: [http://localhost:3000](http://localhost:3000)
   * **API бэкенда (Swagger Docs)**: [http://localhost:8000/docs](http://localhost:8000/docs)

*Примечание: При первом запуске автоматически применятся миграции БД и создастся администратор по умолчанию:*
* **Email**: `admin@emergent.edu`
* **Пароль**: `admin123`

## Локальный запуск в режиме разработки (без Docker)

### 1. Запуск базы данных
Вам потребуется установленный инстанс PostgreSQL. Создайте базу данных `emergent_db`.

### 2. Запуск Бэкенда
1. Перейдите в папку бэкенда:
   ```bash
   cd backend
   ```
2. Создайте файл `.env` на основе примера `.env.example`.
3. Установите зависимости (на Windows потребуется установить системную библиотеку **OpenSlide**):
   ```bash
   pip install -r requirements.txt
   ```
4. Примените миграции:
   ```bash
   alembic upgrade head
   ```
5. Запустите FastAPI сервер:
   ```bash
   uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload
   ```

### 3. Запуск Фронтенда
1. Перейдите в папку фронтенда:
   ```bash
   cd ../frontend
   ```
2. Установите зависимости:
   ```bash
   npm install
   ```
3. Запустите dev-сервер:
   ```bash
   npm run dev
   ```

## Деплой на Railway (Cloud Production)

Приложение спроектировано для развертывания мультисервисной архитектуры на **Railway**:
1. **База данных**: Создайте сервис PostgreSQL в Railway.
2. **Бэкенд-сервис**: Укажите путь к Dockerfile: `backend/Dockerfile`. Передайте переменную `DATABASE_URL` ссылкой на БД (`${{Postgres.DATABASE_URL}}`).
3. **Фронтенд-сервис**: Укажите путь к Dockerfile: `frontend/Dockerfile`. Сгенерируйте домен.
